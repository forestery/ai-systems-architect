# 番外篇：Agentic 软件工程的未来——一种新框架的设定思路

> **思想实验 · 2026**
>
> 前八章讨论了架构模式、范式迁移、Context Engineering 和职业路径。这篇番外篇追问一个更深的问题：如果 Brooks 在《人月神话》中建立的所有假设对 agent 都不成立，我们应该建立什么样的新框架？

---

## 一、人月神话的假设，对 agent 全部失效

Brooks 的框架塑造了几十年软件工程方法论的地基（详见[第 2 章](./chapter_03.md)和第 3 章的讨论）。它的隐含前提是：沟通成本随人数平方增长；人有认知负荷上限，需要模块化和分层抽象；人会疲劳、离职、误解需求，需要文档、流程和 review；知识在人头里，交接是最大风险。

这些约束对 agent **根本不成立**。

| 维度 | 人类工程师 | LLM Agent |
|------|-----------|-----------|
| 沟通成本 | O(n²) 爆炸 | 接近零（共享上下文） |
| 并发能力 | 极低，切换有损耗 | 可无限并行 |
| 知识衰减 | 会遗忘、会过时 | 训练截止前全量保留 |
| 一致性 | 差，个体差异大 | 同 prompt 高度一致 |
| 真正的瓶颈 | 沟通与协调 | 上下文窗口 + 长程连贯性 |
| 真正的失效模式 | 疲劳、政治、懈怠 | 幻觉、漂移、不知道自己不知道 |

约束集不同，方法论就必须不同。用 Scrum、需求评审会、code review 流程来组织 agent——不是效率问题，是方向问题。

## 三、并行替代线性：超越人类规模的可能

旧框架的另一个隐含约束是线性流程——需求→设计→开发→测试，依次推进。这个顺序来自人类的资源限制，而非逻辑必然。

Agent 可以同时跑十个方案，评估后选最优合并。这在人类团队里不可能，成本太高。在 agent 体系里，它是默认操作。「探索空间」本身成为一种设计工具——不再是「找到一个足够好的方案」，而是「在并行候选中找到最优解」。

然而并行解决了规模和成本的问题，却没有解决一个更深的问题：**谁来定义「最优」？** 这引向创造性与主动性的核心矛盾。

---

## 四、创造性与主动性：无法替代，但可以逼近

人类创造性的来源可以拆解：问题重构能力（质疑框架本身而非在框架内优化）、跨域迁移（因生活经验的杂乱性产生意外连接）、对「不舒适感」的感知（说不清为什么，但这里别扭），以及主动性的本质——对现状的内生不满足。

Agent 目前的创造性是重组式的，不是生成式的。它能在已知解法空间里做高质量插值，但不能真正跳出这个空间。更根本的是：**agent 没有立场**。创造性往往来自「我认为这件事应该是这样的」这种主观执念。没有立场，就没有打破现有框架的动力。

以下三条路径是目前最可操作的逼近方式：

> **路径一：红队 agent——让破坏成为结构性设计**
>
> 设计一类 agent，其唯一职责是质疑和破坏：持续提出反例，主动搜索边缘 case 和假设漏洞，不生产代码，只生产「这里可能错了」。这不是创造性，但它在结构上逼迫创造性发生——红队不需要有立场，只需要有破坏的工具和权限。

> **路径二：问题生成替代问题求解**
>
> 当前框架几乎全部精力在 solution space。让 agent 的任务不是「解决这个问题」，而是「找到值得解决的问题」——给 agent 访问代码库、用户反馈、错误日志的权限，让它自主提出工单。这是主动性最可操作的形式，在技术上已接近可行，但没有人把它当第一公民来设计。

> **路径三：人类作为品味提供者，而非任务分配者**
>
> 人类在新框架里的核心价值不是决策每个细节，而是提供无法形式化的判断标准——「这个方案虽然正确但太丑」「这个架构三年后会变成负担」「这个交互完成了任务但用户会觉得奇怪」。这些判断没法写进 spec，但可以通过持续的反馈信号校准 agent 的判断倾向。

## 五、「裁判，不是立法者」：校准评价函数的陷阱

一种直觉上的解法是：把人类的决策过程固化为规则，让 agent 执行规则。这等价于通过 LLM 的能力把人类的决策模式提炼为规则库——本质上是专家系统的老路，换了个皮。

这条路有一个致命问题：提取出来的是**决策的尸体**，不是决策本身。人类判断最有价值的部分恰恰相反——同一个人在不同情境下判断不同，好的判断往往是对规则的例外处理，真正的创造性决策发生在「我也不知道为什么，但就是这样」的时刻。把这些固化成规则，规则覆盖的边界就是系统能力的边界，新情境来了就失效。

更准确的模型需要区分两件事：

- **固化过去的决策模式** → 知识提取，会失效
- **持续参与当下的评价过程** → 实时介入，一次性情境判断，不被泛化

> 人类应该是**裁判**，而不是**立法者**。裁判不需要预先写出所有规则，只需要在球到了的时候做出判断。且这次判断不被泛化为通用规则。

这和第 8 章讨论的 AI Systems Architect 角色高度互补——那个角色不仅是编排 agent，更是在关键节点做一次性情境判断，而不试图把判断转化为可执行的自动化规则。

---

## 六、真正的缺口：agent 的元认知

裁判模型引出了一个结构性困难：**什么时候该停下来问人类？**

如果 agent 问太多，人类成了瓶颈，比原来更低效；如果问太少，关键节点被错误地自动处理，错误积累到不可逆。

这个「何时中断」的判断，需要 agent 理解两件事：

1. 这个决策的后果不可逆性有多高？
2. 这个情境是否超出了我的可信范围？

这是当前 agent 最欠缺的能力——**不是不知道答案，而是不知道自己不知道**。

这个元认知能力一旦出现，人机分工才能真正从「人类固化规则给 agent 执行」，变成「agent 在可信范围内自主，在边界处主动交还人类判断」。在此之前，所谓的「校准评价函数」都只是规则固化的另一种说法。

> 元认知不是功能特性，是**基础设施**。就像操作系统需要知道何时把 CPU 交给内核、何时交还给用户态——agent 系统需要同样的中断判断机制，但它的判断对象不是硬件资源，而是认知边界。

## 八、新框架的人机分工原则

| 层次 | 人类持有 | Agent 承担 |
|------|---------|-----------|
| 问题层 | 问题重构权；质疑框架本身的能力 | 在给定框架内并行探索解法 |
| 判断层 | 品味与价值判断；意义驱动的选择 | 红队压力；主动问题发现与工单生成 |
| 执行层 | 关键节点的一次性裁决（不泛化为规则） | 并行实现；自动化验证；可追溯决策链 |
| 治理层 | 约束、边界与价值观设定 | 可信范围内自主；边界处主动交还控制权 |

---

## 九、结语：判断力才是核心竞争力

新框架的核心命题可以被压缩为三句话：

> **以验证为中心，以意图为锚点，以元认知为基础设施。**

人类不再是最好的执行者，但必须是框架的设定者和品味的校准者——而不是规则的立法者。执行能力不再是核心竞争力。**判断力才是。**

这反过来对人类提出了更高要求：你需要有足够清晰的品味、足够深刻的判断力，才能成为一个有效的裁判。没有这个能力的人，在新框架里会比在旧框架里更快地被边缘化——因为执行这件事，已经完全不是瓶颈了。

---

<p class="closing-quote">
真正在相当长时间内无法逼近的部分，是<strong>对意义的感知驱动行动</strong>。人类会因为觉得「这件事重要」而主动投入超出要求的努力。Agent 没有这个结构——它的努力程度取决于 prompt，而不是内生的意义感。这意味着真正的主动性暂时只能由人类持有，agent 能做的是把人类主动性的覆盖范围和执行效率放大一到两个量级。
<br><br>
人类与 agent 的关系，不是替代，而是放大——前提是人类先想清楚自己在放大什么。
</p>
